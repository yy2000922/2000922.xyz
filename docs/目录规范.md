1) src/_data/：全站“唯一真源”

放全站都会用到的东西：

metadata.js：站点 title/description/url/默认 ogImage/social

navigation.js：导航结构（可选）

constants.js：常量（可选）

原则：模板不要直接读 env，而是让 _data 统一接 env，输出成 metadata.url 之类的字段。

2) src/_includes/：只放“可渲染的东西”

layouts/：页面骨架（base/page/post）

partials/：可复用片段（header/footer/nav/pagination）

macros/：宏（可选）

原则：布局负责结构与 head；内容负责数据；partials 负责复用 UI。

3) src/content/：只放“内容”

posts/：文章（md）

pages/：静态页（md 或 njk）

原则：内容目录里尽量不要塞构建脚本/全局模板，避免耦合。

4) src/static/：只放“必须原样输出的文件”

几乎所有网站都会需要：

robots.txt / favicon / manifest / icons

可能还有 _headers、_redirects（看平台）

原则：这些文件不应该被模板系统“加工”，直接输出最可靠。
