写博客可以像记笔记一样轻松，而不是像部署项目那样复杂。

这里的核心思路是：**写作和发布都在浏览器中完成，但内容仍然直接进入你的 GitHub 仓库**。

## 你会得到什么

- 在浏览器里写作、预览、发布
- 发布动作就是一次仓库提交，可追溯、可回滚
- 写完即发布，无需打开终端

## 简化后的发布流程

1. 在浏览器打开写作入口
2. 用 Markdown 记录内容（就像写笔记）
3. 点击发布，内容提交到 GitHub
4. 站点自动构建并上线

这意味着你既保留了 GitHub 带来的可控性与归档能力，又不用为了发一篇文章去配环境、跑命令。
